# 神魔镇荒 下一阶段开发计划

版本：v0.2  
状态：已完成基础实现
更新时间：2026-05-02  
承接文档：[神魔镇荒 MVP 实现状态](./mvp-implementation-status.md)
实现记录：[神魔镇荒 v0.2 实现状态](./v0.2-implementation-status.md)

## 1. 阶段目标

下一阶段目标不是继续堆大量内容，而是把当前 MVP 打磨成“可以反复试玩的垂直切片”。

这一阶段要回答的问题：

1. 普通战、精英战、首领战是否有明显差异。
2. 士气、建筑生产、阵位、费用、手牌轮转是否足够清楚。
3. 迷宫路线选择是否有策略价值，而不是只是点下一个按钮。
4. 战斗反馈是否足够爽，能不能支撑后续更复杂特效。
5. 配置、调试、测试工具是否足够支撑快速迭代。

阶段完成后，理想状态是：

- 玩家可以完整打一轮迷宫，并愿意再开一局试不同路线。
- 主要战斗规则在画面上能看懂，不需要解释代码逻辑。
- 开发时可以快速跳过战斗、验证奖励、验证首领和路线。
- 内容还不多，但每类内容都有清楚的功能定位。

## 2. 开发原则

- 先手感，后数量。战斗不好玩时，不继续堆卡牌和神器。
- 先可读，后华丽。特效要让玩家看懂攻击来源、命中目标和结果。
- 先调试工具，后复杂系统。能快速复现问题，迭代速度才不会慢。
- 不急着加第二个角色。边境指挥官先打磨到有明确流派。
- 不引入联网、PVP、排行榜和复杂剧情。

## 3. 里程碑

### 3.1 M1：MVP 稳定化

目标：把当前已发现的体验问题修干净，让基础闭环不再卡住。

必须完成：

- 普通怪物节点才有敌方基地。
- 精英和首领节点只保留敌方核心，不再显示敌方基地。
- 精英和首领核心固定在上方阵地，不能直接攻击我方基地。
- 敌方核心通过范围攻击、技能和派兵制造压力。
- 阵位改为限制建筑数量；士兵、精兵、英雄和建筑产物不占阵位。
- 远程单位射程按类型区分，避免普通远程全图输出。
- 士气作用在 UI 上清楚展示。
- 保留一键胜利按钮，方便快速测试迷宫和奖励。

验收标准：

- 精英和首领关不会因为攻击不存在的基地导致无法胜利。
- 玩家能看懂当前士气数量、士气进度和下一张牌是否会被强化。
- 阵位数值只随建筑数量变化，不随士兵、精兵、英雄或建筑产物变化。

### 3.2 M2：战斗体验升级

目标：让战斗从“能跑”变成“有明显对抗感”。

必须完成：

- 敌方核心技能框架：
  - 周期性派兵。
  - 范围攻击我方士兵。
  - 给敌方单位加增益。
  - 生命低于阈值后改变技能节奏。
- 单位定位清晰：
  - 近战兵负责顶线。
  - 远程兵负责后排输出。
  - 精英兵负责突破或抗线。
  - 英雄兵负责高影响力爆发。
  - 建筑负责持续生产和阵地构筑。
- 战斗反馈：
  - 攻击弹道更清楚。
  - 命中特效和受击闪白更明显。
  - 死亡反馈更明确。
  - 首领技能有预警圈或明显前摇。
- 战斗 UI：
  - 显示敌方核心血量。
  - 显示当前目标类型：敌方基地 / 敌方核心。
  - 显示士气强化提示。
  - 调试按钮集中放到一个小调试区。

验收标准：

- 普通战、精英战、小首领战在玩法上能明显区分。
- 玩家能从画面判断为什么某个单位死亡、为什么士气触发。
- 一场战斗内至少有 2-3 次需要玩家主动决策的时刻。

### 3.3 M3：迷宫路线升级

目标：让迷宫不只是节点列表，而是有路线选择压力。

必须完成：

- 每层显示完整路线图，从下往上走。
- 房间之间用线连接。
- 每个房间只连接部分上方房间，不连接全部上方房间。
- 保证任意可选路线都能走到本层终点。
- 当前可走房间高亮。
- 已走路径高亮。
- 未解锁房间置灰。
- 房间悬停或点击前展示节点预览：
  - 节点类型。
  - 可能奖励。
  - 是否为战斗节点。
  - 精英 / 首领的简短提示。

验收标准：

- 玩家能在进入房间前判断这条路线后续大概通向哪里。
- 玩家会为了商店、休息、神器或避开神秘节点做路线选择。
- 路线图不会出现断路或无法到达首领的情况。

### 3.4 M4：奖励与构筑升级

目标：让每次战斗后都有明确的构筑选择。

必须完成：

- 战斗后卡牌奖励改为“从 3 张中选 1 张”。
- 首领奖励改为更高质量的多选奖励。
- 神器层展示 3 个神器选 1 个。
- 商店展示更清楚：
  - 购买区。
  - 出售区。
  - 当前金币。
  - 卡牌等级和类型。
- 休息节点优化：
  - 回血。
  - 三合一合成。
  - 显示可合成卡牌。
- 机遇节点优化：
  - 至少 6 个事件模板。
  - 奖励、风险和文字表现区分开。

验收标准：

- 玩家每次拿卡或拿神器时有选择，而不是直接塞进卡组。
- 商店和休息节点能改变本局构筑，不只是过场。
- 迷宫路线和奖励选择能形成简单流派。

### 3.5 M5：边境指挥官流派打磨

目标：让第一个角色真正有职业味道。

重点流派：

- 兵营召唤流：多建筑、多普通士兵、依靠士气滚雪球。
- 精兵突破流：少量高质量单位，配合护盾和鼓舞。
- 法术控场流：建筑守线，法术清怪和打断首领节奏。

必须完成：

- 士气机制说明和反馈。
- 士气强化效果扩展：
  - 士兵牌：额外召唤。
  - 精英牌：登场护盾。
  - 英雄牌：登场短时间增伤。
- 战鼓法令和士气直接关联。
- 首批卡牌重新梳理到 25-35 张可玩卡，不只追数量。
- 首批神器重新梳理到 15-25 个，覆盖战斗、经济、路线、职业强化。

验收标准：

- 玩家能理解“我这局是走召唤、精兵还是法术控场”。
- 士气不是隐藏数值，而是边境指挥官的核心节奏。

### 3.6 M6：表现与素材升级

目标：在不依赖第三方免费素材包的前提下，让画面从原型进入可展示状态。

必须完成：

- 继续使用 AI 生成素材，并登记到 `docs/ai-asset-register.md`。
- 增加节点图标、神器图标、奖励卡面。
- 补齐至少一套中式神话风格敌人：
  - 普通小怪。
  - 远程怪。
  - 高血量怪。
  - 精英核心。
  - 小首领。
  - 最终首领。
- 攻击特效分层：
  - 普通近战。
  - 普通远程。
  - 法术爆炸。
  - 首领范围技能。
  - 士气强化反馈。
- 简单音效：
  - 出牌。
  - 召唤。
  - 命中。
  - 胜利。
  - 失败。

验收标准：

- 不看文字也能大概分辨单位类型。
- 出牌、命中、死亡、首领技能都有反馈。
- 素材来源和处理记录完整，便于后续 Steam AI 内容披露。

### 3.7 M7：配置和调试工具

目标：让后续调数值和扩内容不靠硬编码。

必须完成：

- 战斗调试面板：
  - 一键胜利。
  - 一键失败。
  - 加金币。
  - 抽卡奖励。
  - 加士气。
  - 跳到下一节点。
- 内容校验扩展：
  - 卡牌等级链完整。
  - 奖励池不为空。
  - 精英和首领必须有核心单位。
  - 普通战必须有敌方基地目标。
  - 所有节点都能到达本层终点。
- 配置整理：
  - 保留 ScriptableObject 作为编辑期配置。
  - 评估是否增加 CSV/JSON 导入工具。
  - 输出卡牌、单位、神器、遭遇的字段清单。

验收标准：

- 修改数值后可以一键校验内容。
- 开发者能快速跳到指定流程验证问题。
- 新增卡牌和神器时不需要改多处硬编码。

## 4. 优先级清单

P0：必须先做

- 精英和首领目标规则稳定。
- 士气 UI 和战鼓效果清晰。
- 阵位只限制建筑数量，士兵不再卡住出牌。
- 一键胜利、基础调试按钮。
- 完整层路线图和路径可达性。

P1：阶段核心

- 首领技能框架。
- 战斗后三选一卡牌奖励。
- 神器三选一。
- 商店和休息节点体验优化。
- 单位射程、攻速、血量、费用的第一轮平衡。

P2：展示增强

- 攻击特效升级。
- 死亡特效。
- 节点和神器图标。
- 简单音效。
- 更多 AI 素材。

P3：暂缓

- 第二个角色。
- 第二个迷宫。
- 完整局外天赋树。
- 图鉴。
- 存档槽管理。
- Steam 成就。

## 5. 建议开发顺序

1. 修完当前战斗目标和士气可读性问题。
2. 完成战斗调试面板。
3. 完成整层迷宫路线图和节点预览。
4. 做战斗后三选一卡牌奖励。
5. 做神器三选一。
6. 做首领技能框架。
7. 重调第一批卡牌、单位、神器数值。
8. 增加表现反馈和音效。
9. 扩展内容校验和自动测试。
10. 做一次完整三层通关体验回放，记录问题再进入下一轮。

## 6. 本阶段暂不做

- 不做多角色。
- 不做多迷宫。
- 不做联网和排行榜。
- 不做复杂剧情。
- 不做完整商业级 UI。
- 不做 Steam 打包发布流程。

这些内容放到 v0.3 或更后面，等第一个角色和第一个迷宫真正好玩之后再扩。
