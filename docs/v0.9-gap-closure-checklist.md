# 神魔镇荒 v0.9 补漏收口清单

更新时间：2026-05-03

## 目标

本阶段不再继续横向堆玩法，而是把已经做出来的 MVP 闭环收紧，重点解决：

1. 规则已经改了，但测试、日志、文档还停留在旧假设。
2. 玩法已经能跑，但 UI 和反馈还没有把“卡牌肉鸽”的感觉打出来。
3. 已知兜底逻辑仍在长期占位，后续会影响打包、调试和继续扩内容。

完成后，理想状态是：

- 当前版本的测试结论可信。
- 迷宫、奖励、事件、商店和休息节点的记录与实际流程一致。
- 战斗 BGM、卡图、牌堆和奖励界面不再像临时原型。
- 文档能真实反映当前实现状态，而不是停在旧阶段。

## P0：必须先收口

### 1. 修正测试与现实现状不一致的问题

目标：

- 让 EditMode 测试重新成为可信基线。

必须修改：

- `MapGenerationServiceTests`
  - 固定房间行应断言为单房间，而不是 `2-3` 个房间。
  - 第 1 / 5 / 10 行的断言要和当前路线生成规则一致。
- `MvpValidationServiceTests`
  - 保持能覆盖当前三职业、固定房间、特殊节点和终局闭环。
- 如有必要，补 1 个针对“固定房间 + 可达性”的回归测试，避免后续改路线图时再倒退。

验收标准：

- EditMode 测试全部通过。
- 测试描述与当前路线图规则完全一致。

### 2. 修正事件日志房间坐标漂移

目标：

- 商店、休息、神器、机遇、神秘、卡牌奖励结算的日志，要记录到“当前完成的房间”，而不是“推进后的下一房间”。

必须修改：

- `GameFlowController`
  - `ResolvePendingNonBattle`
  - `FinishPendingCardReward`
  - `AppendRunLog`
- 结算日志需要基于被结算的 `pendingNode` 或显式传入的房间坐标生成，不再直接读已经推进后的 `CurrentRun.floor/row`。

验收标准：

- 进入休息房后回血或合成，日志仍显示刚才的休息房坐标。
- 战斗后选卡、跳过奖励、神秘事件奖励等记录不串行、不跳格。

### 3. 彻底收口战斗 BGM 资源加载

目标：

- 停止长期依赖程序生成临时 BGM。

必须修改：

- `BattleController`
  - 优先改成显式资源引用或更稳定的资源装载方式。
  - 保留临时兜底，但只能作为真正异常场景的最后保护，不再作为日常正常路径。
- `docs/known-issues.md`
  - 同步更新为“已解决”或记录剩余限制。
- `docs/third-party-asset-register.md`
  - 如果音频格式、路径或导入方式变化，要同步登记。

验收标准：

- Editor 内进入战斗时能稳定播放外部 BGM。
- Windows 打包后也能播放同一首 BGM。
- Console 不再常态输出“使用程序生成的临时战斗 BGM”。

### 4. 修正手牌卡图与牌堆表现的根因

目标：

- 解决手牌卡图顶部镂空、卡图未贴满牌框、牌堆偶发发黑这些反复出现的问题。

必须修改：

- `BattleUiController`
  - 手牌卡图区域改为按牌框裁切后的“铺满优先”策略，不再使用当前的完整塞入式缩放。
  - 手牌中的卡图旋转要跟随卡框视觉，而不是始终正向矩形摆放。
  - 牌堆预览图为空时，要有稳定可读的卡背，不允许出现发黑或像丢图的观感。
  - 卡池 / 已用卡池 / 神通按钮 / 基地信息的位置关系重新过一轮，避免再次互相遮挡。

验收标准：

- 手牌中所有卡图都能贴合牌框主要展示区域。
- 牌堆在有图和无图两种情况下都稳定可读。
- 神通、基地血量、卡池、已用卡池不会相互遮挡。

## P1：直接提升成品感

### 5. 把奖励、商店、休息做成真正的卡牌构筑界面

目标：

- 让“选卡、买卡、卖卡、合成”看起来像肉鸽卡牌游戏，而不是调试按钮面板。

必须修改：

- `MainMenuController`
  - 卡牌奖励面板显示卡图、类型、费用、等级和核心描述，不只是一块文字按钮。
  - 商店区分购买区、出售区、净化区，卡面样式统一。
  - 休息节点的可合成卡牌也用卡面展示，而不是纯文本按钮。
  - 神器、事件、路线图的视觉样式继续统一到“神魔镇荒”的当前美术方向。

验收标准：

- 玩家在不看说明文档的情况下，也能区分“奖励选卡”“商店购买”“休息合成”。
- 卡图和数值信息成为主视觉，不再主要依赖长段文本。

### 6. 补一组关键流程回归测试

目标：

- 覆盖当前最容易反复出问题的流程，而不只测静态配置。

建议补充：

- `GameFlowController` 级别测试
  - 非战斗节点结算后房间推进是否正确。
  - 卡牌奖励选择 / 跳过 / 换金币后的状态是否正确。
  - 商店刷新、购买、出售、净化是否按次数和金币限制工作。
  - 神器刷新次数与金币扣除是否正确。
- 英雄唯一性回归
  - 英雄出战后移出本场牌循环。
  - 英雄阵亡后不会卡手，也不会再次洗回。

验收标准：

- 关键流转问题可以在测试里复现，不再完全依赖手工试玩。

## P2：结构与文档清理

### 7. 清理误导性旧结构

目标：

- 减少“项目里还留着旧原型入口或旧控制器”的误导。

建议处理：

- 评估 `RoguelikeRunController` 是否仍有实际用途。
  - 如果没有实际引用，删除或明确标记为旧原型残留。
- 检查是否还有只服务于早期原型的无效兜底或重复入口。

验收标准：

- 代码里不存在明显和当前主流程并行、但实际已废弃的迷你控制器。

### 8. 同步 README 与状态文档

目标：

- 让新成员或未来自己打开仓库时，看到的不是过期版本描述。

必须修改：

- `README.md`
  - 不再写“当前目标是 v0.2 垂直切片”。
  - 改成当前版本已经具备的三职业、三层探索、路线图、奖励、神器、事件等现状。
- `docs/mvp-implementation-status.md`
  - 从“边境指挥官单角色视角”改为真实反映当前三职业和后续阶段。
- 如有必要，新增一份“当前版本说明”，集中描述：
  - 已完成
  - 已知临时方案
  - 下一步重点收口项

验收标准：

- README、状态文档、当前实现三者一致。
- 不再出现“文档写 v0.2，代码已到 v0.8”的阶段漂移。

## 本阶段不新增

- 不新增第四个职业。
- 不新增第二个迷宫。
- 不新增大批新卡牌或新神器。
- 不重做战斗规则大框架。
- 不追求商业最终美术质量。

## 建议执行顺序

1. 先修测试基线和日志坐标。
2. 再修 BGM 真正加载链。
3. 然后集中处理手牌卡图、牌堆、神通和基地 HUD 的遮挡与铺图问题。
4. 接着升级卡牌奖励 / 商店 / 休息的卡牌化 UI。
5. 最后做旧结构清理和文档同步。

## 完成定义

本清单完成后，当前版本应达到：

- 规则可信：测试、日志、文档都与实现一致。
- 体验可信：奖励、卡图、牌堆、BGM 不再明显像临时兜底。
- 迭代可信：后续继续补肉鸽深度时，不需要反复回头修同一类基础问题。
