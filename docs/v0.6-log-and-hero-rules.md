# v0.6 日志与英雄规则开发记录

## 本次目标

继续补齐肉鸽 MVP 的可读性和战斗规则细节：玩家需要能回看本局发生了什么，也需要在出牌前看懂卡牌为什么不能释放。

## 已完成功能

### 1. 本局事件日志

- `RunState` 新增本局事件日志列表。
- `GameFlowController` 在进入房间、战斗结算、获得卡牌、购买、出售、刷新商店、净化卡牌、刷新神器等节点写入日志。
- 迷宫左侧菜单新增 `日志` 面板，展示最近的本局记录。
- 日志措辞使用 `迷宫 N · 房间 M/10`，避免回到早期“第几层第几行”的表达。
- 日志只保留最近若干条，避免长局中 UI 和存档状态无限增长。

### 2. 英雄唯一性

- 英雄牌现在会检查同名英雄是否已经在场。
- 如果该英雄仍然存活，手牌会置灰，并提示 `该英雄已经在场`。
- 这样可以避免重复拖出多个同名英雄，保留“英雄单位是强力唯一单位”的定位。

### 3. 卡牌禁用原因

- 手牌不能释放时，不再只显示笼统的 `费用或建筑位不足`。
- 现在会区分：
  - `费用不足`
  - `建筑位不足`
  - `该英雄已经在场`
- 鼠标拖拽灰色卡牌时，会弹出对应原因，降低试玩时的理解成本。
