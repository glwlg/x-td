# v0.5 构筑可读性与战斗效果开发记录

## 本次目标

继续把 MVP 往“能反复试玩”的方向推进：玩家在迷宫里能随时看懂自己的卡组、神器、成长和内容规模；战斗里的控制、持续伤害、经济牌也开始真正生效。

## 已完成功能

### 1. 迷宫侧边信息面板

左侧菜单现在可以实际点击：

- `路线`：关闭侧边详情，回到完整路线图。
- `卡组`：查看当前卡组数量、类型分布、可合成数量和主要卡牌列表。
- `神器`：查看本局神器和永久神器。
- `成长`：查看当前探索状态、永久探索次数、通关次数、总经验、主角等级和本层词缀。
- `图鉴`：查看当前内容规模，包括卡牌、神器、敌我单位和遭遇数量。

这一步主要解决“肉鸽构筑看不清”的问题。

### 2. 战斗状态补齐

现在底层战斗支持更多效果：

- `燃烧`：持续造成伤害。
- `中毒`：持续造成伤害。
- `击退`：把非首领、非建筑单位推离目标点。
- `战斗内获得金币`：经济牌可以直接影响本局金币。

此前已有的减速和眩晕也继续保留。

### 3. 新增卡牌

新增并接入奖励池/商店池：

- `瘴毒葫芦`：目标区域中毒并减速。
- `推山印`：目标区域击退并造成少量伤害。
- `聚宝符`：获得金币并抽 1 张牌。

同时增强已有卡牌：

- `三昧真火` 2 级以上会附带燃烧。
- `掌心雷符` 3 级会附带击退。

### 4. 可读性调整

- 左侧菜单文案改为更直接的 `路线 / 卡组 / 神器 / 成长 / 图鉴`。
- 这些面板暂时是运行时 UI，后续如果做正式 UI 资源，可以迁移到 prefab 或 UI Toolkit。

## 验证记录

已使用 Unity Roslyn 编译检查，结果通过。

- `UiFontProvider.cs` 在命令行检查里被临时重复传入。

这两个警告不是本次功能引入的编译错误。
