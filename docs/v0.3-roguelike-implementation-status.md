# v0.3 肉鸽系统开发记录

## 本次目标

本阶段补齐 MVP 中“还没有肉鸽味”的部分，让一次探索不只是连续打战斗，而是开始有构筑选择、风险事件、诅咒负担和每层规则变化。

## 已完成功能

### 1. 战后奖励三选一

- 普通怪物战斗胜利后，现在会出现 3 张卡牌，选择 1 张加入卡组。
- 精英、小 Boss、最终 Boss 仍然保留更高规格的卡牌奖励。
- 普通战斗的卡牌奖励支持“放弃剩余奖励换金币”的运行状态字段，UI 点击时会走对应结算逻辑。

### 2. 卡牌稀有度和权重

- 奖励抽取不再是简单随机，而是按节点奖励等级走权重：
  - 普通战斗：以普通卡为主，小概率出现高稀有度。
  - 精英战斗：提高非普通卡权重。
  - Boss/神秘高收益：明显提高稀有、史诗、传说卡权重。
- 奖励池默认排除诅咒卡，诅咒只通过事件或惩罚进入卡组。
- 新增高稀有卡：
  - `星河诏令`
  - `盘古残火`
  - `哪吒点兵`
  - `天工营盘`

### 3. 神器影响构筑

新增更偏构筑方向的神器：

- `万兵旗`：建筑牌费用降低，建筑产兵更快，但法术牌费用提高。
- `雷火符匣`：法术牌费用降低，法术伤害提高，但建筑牌费用提高。
- `点将台`：每打出 1 张建筑牌，立刻获得 1 点士气。
- `镇煞葫芦`：打出诅咒牌时不再受罚，改为获得 1 点士气。

这些效果已经接入战斗中的费用计算、建筑产兵间隔、法术伤害、士气触发和诅咒结算。

### 4. 机遇/神秘事件

- 机遇节点不再是单按钮开奖，现在会显示 3 个可选事件。
- 神秘节点不再进入房间就自动随机结算，现在会显示 3 个处理方式：
  - `稳妥探查`：稳定获得金币和卡牌奖励。
  - `深入凶阵`：可能获得高收益，也可能触发惩罚战。
  - `献祭换宝`：失去生命并加入诅咒，优先换取神器。

### 5. 诅咒/负面卡

新增 3 张诅咒牌：

- `业火缠身`：打出后灼伤我方基地。
- `妖雾侵心`：打出后扣除当前费用。
- `因果债契`：打出后引来额外妖兵。

诅咒牌占手牌和卡组位置，可以拖出清掉，但会触发负面效果；如果拥有 `镇煞葫芦`，诅咒会被转化为士气。

### 6. 每层主题词缀

每次探索开始时会为 3 层各生成一个主题词缀：

- `妖雾迷天`：远程射程下降，法术费用 +1。
- `天雷劫池`：战场周期性落雷，法术伤害小幅提高。
- `万妖潮`：敌方出兵更快，金币收益 +15%。
- `仙阵余辉`：我方建筑更硬且产兵略快，但费用恢复略慢。

当前层词缀会显示在路线图上，并在战斗中实际生效。

## 验证记录

已使用 Unity 自带 Roslyn 编译器做运行时代码编译校验，结果通过。

命令行校验时需要临时显式带上 `Assets/Scripts/Presentation/UiFontProvider.cs`，因为当前 `Library/Bee` 的响应文件还没有刷新到这个新增脚本。Unity 编辑器重新导入脚本后，这个临时补充通常会消失。

编译结果只剩 3 个 Unity 过时 API 警告，和本次 v0.3 功能无关。

## 后续建议

- 在 Unity 里实际跑一局，确认普通战斗胜利后卡牌奖励面板、神秘节点选择、诅咒牌拖出都能按预期流转。
- 下一步可以做“事件日志”和“本局构筑摘要”，让玩家知道这一局为什么变强或变糟。
- 后续要继续把神器和词缀做得更夸张，比如专门支持召唤流、法术流、建筑流、诅咒流。
