# 神魔镇荒 MVP 实现状态

更新时间：2026-05-03

## 已完成闭环

- 战斗：上下正面对抗、拖拽出牌、费用回复、统率限制、单位自动推进/索敌/攻击、建筑持续生产士兵、弹道/命中特效/伤害数字、胜利失败结算。
- 卡牌：卡池、手牌、已用卡池、手牌打空后重新抽满、建筑/普通士兵/精英/英雄/法术/战术牌、1-3 级卡牌、三合一升级。
- 迷宫：3 层，每层 10 行；整层房间由下往上展示，分支部分连线且保证能走通；第 1 行普通战斗，第 5 行精英，第 10 行首领；第 5 行后出现商店、休息、机遇、神器和神秘节点，并带有更随机的分布权重。
- 节点：普通怪物、精英、商店、休息、机遇、神秘、神器、小首领、最终首领。
- 奖励：金币、战斗后卡牌、商店购买、半价出售、机遇奖励、神器选择、小首领经验、最终首领永久神器。
- 角色：边境指挥官、万灵召主、雷火术士三职业都已接入初始卡组、职业神通和差异化构筑方向。
- 战斗特性：建筑不占建筑位以外的额外统率；英雄牌本场唯一，出战后会移出循环，阵亡后不会卡手。
- 首领机制：精英与首领不再附带额外敌方基地，而是以核心单位站桩输出、范围技能与派兵构成压力。
- UI：战斗 HUD 已拆分卡池/已用牌堆、神通按钮、基地信息与首领提示区；奖励、商店、休息节点已卡牌化展示。
- 内容量：33 张卡牌、18 个神器、普通怪物池、3 个精英、2 个小首领、1 个最终首领。
- 局外成长：主角累计经验、通关次数、永久神器通过 `PlayerPrefs` 保存。
- v0.9 收口：房间结算日志改为记录已完成房间；战斗 BGM 增加 `Resources -> StreamingAssets -> 程序生成` 的三级加载链；新增 `GameFlowControllerFlowTests` 作为流程回归测试基线。历史增强项详见 [神魔镇荒 v0.2 实现状态](./v0.2-implementation-status.md)。

## 当前简化

- 战斗仍是单主战线，没有多路线和格子摆放。
- 首领技能先以核心单位 + 召唤压力表达，没有完整阶段技能。
- 神秘节点先实现为高收益奖励或无奖励强力战斗。
- 永久神器先做轻量数值效果，后续再扩展为更多局外选择。
- 美术以 AI 生成素材和本地处理素材为主，尚未达到最终商业美术质量。
