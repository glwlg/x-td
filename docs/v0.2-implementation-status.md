# 神魔镇荒 v0.2 实现状态

更新时间：2026-05-02  
版本目标：可反复试玩的垂直切片

## 总体状态

v0.2 已在现有 MVP 基础上补齐战斗差异、路线预览、奖励选择、机遇模板、调试工具、校验规则和基础反馈。当前版本仍然是原型/垂直切片，不代表最终商业美术质量，但核心玩法闭环已经可以用于反复试玩和调数值。

## M1：MVP 稳定化

状态：完成

- 普通怪物节点保留敌方基地，胜利目标是摧毁基地。
- 精英、小首领、最终首领节点改为敌方核心，隐藏敌方基地。
- 敌方核心固定在上方，不直接攻击我方基地。
- 阵位改为限制建筑数量，建筑生产出的士兵不再占阵位。
- 士气 UI 显示当前士气、进度和下一张出兵牌强化效果。
- 保留并扩展调试按钮，方便快速测试胜利、失败、奖励和路线推进。

## M2：战斗体验升级

状态：完成

- 敌方核心加入技能框架：
  - 周期性范围技能。
  - 技能落点有预警圈。
  - 周期性强化敌方小兵。
  - 核心血量低于 50% 后进入狂暴，派兵和技能节奏加快。
- 士气强化按单位类型区分：
  - 士兵牌额外召唤 1 个单位。
  - 精兵牌登场获得护盾。
  - 英雄牌登场获得短时间增伤。
- 出牌、召唤、命中、死亡、胜利、失败都有基础反馈。
- 战斗 UI 增加集中调试区。

## M3：迷宫路线升级

状态：完成

- 每层展示完整路线图，从下往上选择房间。
- 房间之间用线连接，每个房间只连接部分上方房间。
- 当前可走房间高亮，已走路径高亮，未解锁房间置灰。
- 房间支持悬停/点击预览，展示节点类型、奖励、风险和战斗提示。
- 内容校验会检查所有房间都能抵达本层终点。

## M4：奖励与构筑升级

状态：完成

- 精英战奖励改为从 3 张卡中选 1 张。
- 小首领奖励改为从 4 张卡中选 2 张。
- 最终首领奖励改为从 5 张卡中选 3 张，并掉落永久神器。
- 神器层保留 3 选 1，拥有观星镜时可选数量增加。
- 商店展示购买区、出售区、当前金币、卡牌类型和等级。
- 休息节点展示回血或三合一合成。
- 机遇节点扩展为 6 个事件模板，明确收益和风险。

## M5：边境指挥官流派打磨

状态：完成

- 当前卡池维持 33 张卡牌，覆盖生产建筑、直接出兵、精兵、英雄、法术和战术。
- 当前神器池维持 18 个神器，覆盖战斗、经济、路线、休息、职业强化和永久奖励。
- 边境指挥官的核心节奏从“隐藏数值”改为 UI 可读的士气资源。
- 战鼓法令直接提供士气，并同时提供短时间攻击增益。

## M6：表现与素材升级

状态：完成

- 继续使用 `Assets/_Project/Art/AI` 下的 AI 生成素材，不再引入免费素材包。
- 手牌、战场单位、背景、弹道、命中特效、法术落点特效已接入运行时。
- 死亡、士气、首领预警使用对象池特效生成，避免战斗中高频创建销毁。
- 简单音效由运行时代码生成，覆盖出牌、召唤、命中、胜利和失败。
- 素材和处理方式记录在 `docs/ai-asset-register.md`。

## M7：配置和调试工具

状态：完成

- 战斗调试面板包含：
  - 胜利。
  - 失败。
  - 加金币。
  - 打开奖励抽卡。
  - 加士气。
  - 跳过当前节点。
- MVP 内容校验扩展：
  - 卡牌数量和 1-3 级链。
  - 神器数量和奖励池。
  - 普通战敌方基地规则。
  - 精英和首领核心规则。
  - 敌方派兵配置。
  - 迷宫路线可达性。
- 配置字段清单见 `docs/content-config-fields.md`。

- Unity 菜单：`神魔镇荒 > 验证 > MVP 内容校验`
- EditMode 测试：
  - `DeckRuntimeTests`
  - `MapGenerationServiceTests`
  - `MoraleTrackerTests`
  - `MvpValidationServiceTests`
- 手动编译检查：
  - Runtime 编译通过，仅有 Unity `FindFirstObjectByType` 过时警告。
  - EditMode 测试程序集编译通过。

## 后续建议

v0.3 应优先做三件事：

1. 用真实试玩反馈调数值，尤其是敌方核心技能频率、阵位上限、费用恢复和奖励密度。
2. 继续补 AI 素材，把敌人、神器、节点图标和技能特效做出更清晰的视觉差异。
3. 把 ScriptableObject 配置整理成可导入表格，减少后续扩卡和调平衡时的硬编码成本。
